Receiving a message on all service instances in MassTransit

19 November, 2019
MassTransit is an excellent distributed application framework for .NET that I intend to write more about later. This post documents a solution to a problem you’ll probably come across when developing distributed applications with MassTransit. MassTransit is built upon a message broker, usually RabbitMQ or Azure Service Bus. MassTransit “consumers” are attached to message broker queues, to which messages are delivered according to the rules of the message broker. One thing about this that’s handy is messages put into a queue are delivered to a consumer only once,1 which means you can start multiple instances of a service and, since they’ll be using the same queue name, you’ll get load balancing “for free”.
